# Track 2 - 로프 액션 (Momentum Swing)

-   **상태:** DRAFT
-   **버전:** 0.2
-   **Sprint:** Sprint 1 - Core Movement Prototype

------------------------------------------------------------------------

# 목적

플레이어가 로프를 이용해 **모멘텀을 생성·유지·방출**하는 이동 시스템을
구현한다.

이 시스템의 핵심은 목적지까지 이동하는 것이 아니라 **속도를 만들어내는
것**이다. 플레이어는 스윙으로 점점 더 큰 속도를 만들고, 원하는 순간
로프를 해제하여 그 모멘텀을 이동과 전투에 활용한다.

------------------------------------------------------------------------

# 플레이어 경험

-   원하는 지형에 즉시 로프를 연결한다.
-   로프가 팽팽해지는 순간 강한 원심력을 느낀다.
-   스윙을 반복할수록 속도가 자연스럽게 증가한다.
-   WASD와 카메라를 이용해 회전 궤적을 능동적으로 조절한다.
-   원하는 순간 로프를 해제하여 현재 속도를 거의 그대로 유지한 채
    날아간다.
-   이후 로켓 반동과 자연스럽게 이어져 더욱 큰 모멘텀을 만든다.

------------------------------------------------------------------------

# 핵심 설계 철학

## Momentum First

-   이동보다 속도 생성이 우선이다.
-   플레이어는 지속적으로 모멘텀을 축적한다.
-   로프는 속도를 제한하지 않는다.
-   로프 해제 시 현재 속도를 그대로 유지한다.

## Rope Tension

로프에는 항상 두 상태가 존재한다.

-   Slack : 장력이 없어 자유낙하와 비슷하다.
-   Tight : 장력이 생기며 원운동이 시작된다.

재미는 Tight 상태에서 발생한다.

------------------------------------------------------------------------

# 상태

``` text
Ready
  ↓
Firing
  ↓
Connected
  ├─ Swing
  ├─ Rope Length Change
  └─ Release
```

Connected는 단순히 매달리는 상태가 아니라 모멘텀을 만드는 상태이다.

------------------------------------------------------------------------

# 입력

  입력        기능
  ----------- -------------------------
  좌클릭      로프 발사(클릭 시작) / 유지(클릭 유지) / 해제(클릭 해제)


------------------------------------------------------------------------

# 로프 길이

길이가 짧아질수록 회전 반경이 줄어들며 더 높은 속도를 얻는다.

------------------------------------------------------------------------

# 로프 발사

좌클릭 입력 시, 마우스의 위치로 (카메라의 중앙으로) 로프 앵커가 발사된다.
좌클릭을 유지하면 로프가 고정되고, 로프 액션을 할 수 있다.
좌클릭 해제 시, 로프가 해제된다. 그러나 속도는 변하지 않는다.

------------------------------------------------------------------------

# 스윙 규칙

-   플레이어는 로프 길이를 넘을 수 없다.
-   로프는 플레이어를 강제로 끌어당기지 않는다.
-   중력과 기존 속도를 유지한다.
-   접선 방향 속도를 최대한 보존한다.
-   원운동이 자연스럽게 이어져야 한다.
-   로프는 속도를 제거하지 않는다.

------------------------------------------------------------------------

# WASD

WASD는 이동이 아니라 스윙을 보조한다.

-   W : 진행 방향 유지
-   S : 감속
-   A/D : 회전 방향 미세 조절

공중 제어는 약하게 유지한다.

------------------------------------------------------------------------

# 로프 해제

-   현재 속도를 유지한다.
-   접선 방향으로 날아간다.
-   추가 추진력은 없다.
-   즉시 다른 로프를 발사할 수 있다.

------------------------------------------------------------------------

# 왼손 : 로프를 잡고 있음

왼손에서 로프가 나간다. Rope origin은 왼손이며, LeftHandCube는 왼손을 담당하는 오브젝트이다.
줄의 앵커가 나가면, 왼손은 앵커쪽을 바라본다. 앵커가 없으면 기본 상태로 돌아온다. 

------------------------------------------------------------------------

# 로프 길이 감소

로프가 연결되어 있을 때, 스페이스 바를 누르면 로프 앵커 쪽으로 추진력을 얻는다.
플레이어가 로프 앵커쪽으로 끌어당겨지며, 로프의 길이가 감소되고 추진력을 얻는다.

------------------------------------------------------------------------

# 구현 원칙

-   Rigidbody 기반
-   Constraint 기반 로프 물리
-   Velocity 직접 제어
-   SpringJoint에 의존하지 않는다.
-   물리와 시각(LineRenderer)을 분리한다.

------------------------------------------------------------------------

# 다른 시스템과의 연동

## Player Movement

-   기존 점프 및 중력 사용
-   공중 이동 가능
-   로프가 모멘텀을 우선적으로 결정

## Rocket Launcher

-   로켓 반동은 로프 속도와 합산된다.
-   두 시스템 모두 Velocity를 공유한다.
-   어느 시스템도 상대의 속도를 초기화하지 않는다.

------------------------------------------------------------------------

# 완료 조건

-   플레이어가 스윙으로 속도를 만들 수 있다.
-   로프 길이를 조절할 수 있다.
-   스윙 중 모멘텀이 자연스럽게 증가한다.
-   해제 시 현재 속도가 유지된다.
-   로켓 반동과 자연스럽게 연결된다.
